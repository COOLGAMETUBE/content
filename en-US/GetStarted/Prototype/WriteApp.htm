Write your app

Now that you have the tools you need, it's time to write your first app. IoT Core is a Universal Windows Platform (UWP)-centric OS and UWP apps are its primary app type. UWP apps are a common app platform across all versions of Windows 10 including Windows 10 IoT Core. The primary tool for writing UWP apps is Visual Studio.

For this section, pick either a C# or a C++ sample app to work with. Then we'll go over three parts to master: 1) Create a project, 2) Deploy a project, and 3) Debug a project.

Create a project

To create a project, start the Visual Studio IDE of your choice – we recommend Visual Studio 2017 – and create a new project from the file tab. In the New Project dialog, navigate to Universal and select the Blank App template


Deploy a project

C++ C#
  1. With your application open in Visual Studio, set the architecture in the toolbar dropdown based on the device you're working with.
  2. In the Visual Studio toolbar, click on the Local Machine dropdown and select Remote Machine.
  3. Right click on your project in the Solution Explorer pane. Then select properties.
  4. Under Configuration Properties -> Debugging, modify the following fields:
    • Machine Name: If you previously used PowerShell to set a unique name for your device, enter the name in Machine Name. Otherwise, use the IP address of your device.
    • Authentication Mode: Set this to Universal (Unencrypted Protocol).
    Then hit the Debug button.
  1. With your application open in Visual Studio, set the architecture in the toolbar dropdown based on the device you're working with.
  2. In the Visual Studio toolbar, click on the Local Machine dropdown and select Remote Machine.
  3. At this point, Visual Studio will present the Remote Connections dialog. If you previously used PowerShell to set a unique name for your device, enter that name in Address. Otherwise, use the IP address of your device.
  4. Set the Authentication mode to Universal (Unencrypted Protocol). Then hit the Debug button.

Debug a project

  1. First, setup the IoT Core console application in Visual Studio. You can use your recently deployed app to copy your binaries or you can do this manually.
  2. To start the remote debugger on your device, connect to the Windows Device Portal and go to the Debug settings section of the Portal. Click Start under Visual Studio Remote Debugger.
  3. The project configuration needs to be modified to enable deployment. To do this, navigate to the Configuration Manager from the Solution Configuration drop-down menu on the toolbar.
  4. From the configuration manager, ensure that the Deploy checkbox is selected for your project configuration. Then hit the Debug button.
